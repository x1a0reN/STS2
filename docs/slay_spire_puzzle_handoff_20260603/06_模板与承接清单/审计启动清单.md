# 审计启动清单

## 输入完整性

审计前确认以下信息全部存在：

- 角色
- 初始生命
- 初始能量
- 每回合抽牌数
- 起始牌组选择范围
- 药水池与选择上限
- 遗物池与选择上限
- 敌人生命
- 敌人每回合行动
- 敌人护甲是否保留
- 状态牌进入玩家哪个牌堆
- 回合上限
- 胜利条件
- 失败条件
- 所有卡牌数量、费用、效果
- 所有改动牌是否标记“（改）”
- 代表解法族

缺任一项，结论只能写“不完整”，不能写通过。

## 红线检查

- 是否存在核心必选。
- 是否把玩家要识别的资源包提前框死。
- 是否默认敌人有手牌、抽牌堆或弃牌堆。
- 是否用固定优先级自动出牌代替自由出牌。
- 是否凭空创造原游戏不存在的关键词、状态或资源系统。
- 是否用单个无竞争药水充当可选资源。
- 是否存在第 1 回合无压力白给窗口。
- 是否所有代表解法都挤在同一个最优回合。

## 枚举要求

最低要输出：

- 合法牌组总数。
- 可成功牌组数。
- 100% 成功路线数。
- 每个牌组的 kill vector。
- 第 2 回合最高成功率。
- 第 3 回合最高成功率。
- 第 4 回合最高成功率。
- 最高成功率路线。
- 最高成功率路线归属解法族。
- 不属于预期解法族的高成功路线数量。

## 判定口径

通过：所有高成功路线都能归入预期解法族，没有 100% 无脑线，没有又快又稳统治路线。

可接受：存在同族更优变体，但不破坏难度和概率梯度。

需调整：代表解法被大量覆盖、快解过稳、主解和稳线失去区分、干扰牌变成主轴。

不通过：100% 稳定线、无脑高成功、不理解核心机制也能过、题面目标和审计结果完全不一致。
