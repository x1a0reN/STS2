## 2. 模拟口径

脚本使用的是精确状态搜索，不是只看单次 Monte Carlo。

计算口径：
1. 玩家每个分支都按“优先更早击杀”的最优策略行动。
2. 敌方 Vulnerable 只影响攻击牌伤害，按 1.5 倍向下取整。
3. Fire Potion 视为直接伤害，不吃 Vulnerable 增伤。
4. 敌方 Vulnerable 在玩家回合结束后减少 1。
5. 对推荐三解，额外做了 20000 次 Monte Carlo 抽样，作为和精确结果的交叉验证。

## 4. 全池审计

精确审计范围：
1. 对 `7-9 张牌` 的所有唯一牌组做精确搜索。
2. 每个牌组都遍历 `无 / Lantern / Anchor` 三种遗物状态。
3. 每个牌组都遍历 `无 / Fire Potion / Strength Potion / Block Potion` 四种药水状态。
4. 合计精确审计 `660` 个资源配置。

关键发现：
1. 找到 `20` 条理论 `100%` 稳定路线。
2. 找到 `11` 条存在首回合击杀分支的路线。
3. 最强反例就是一个 `7 张牌` 组合：
   `Bash, Defend, Perfected Strike, Strike x3, Uppercut + Lantern + Fire Potion`
4. 该反例的精确结果是：
   第 1 回合 47.6190%，第 2 回合 52.3810%，最终失败 0.0000%。

这意味着什么：
1. 玩家少选到 7 张后，抽牌稳定性被大幅抬高。
2. `Lantern + Fire Potion` 在这道题里过强。
3. 出题人虽然给了三条“看起来合理”的推荐解，但完整资源池里已经存在更强、更稳、更快的偷跑线。

## 5. 设计者判定

结论：
这道题目前 **不算合格残局**，不能定稿。

理由：
1. 反例审计已经找到了多条 `100%` 稳定路线，直接违反“推荐或非推荐资源配置后仍应保留失败空间”的要求。
2. 少选牌组会显著提升稳定性，说明题面对“薄牌逃课”约束不够。
3. `Fire Potion + Lantern` 的组合把题目从“教学残局”推成了“找到爆发包就稳过”的半显性钥匙题。
4. 推荐三解虽然形式上做出了 `1 / 2 / 3 回合` 梯度，但完整资源池下的真实最强路线已经压过了这个梯度。

## 6. 下一步修正方向

如果要把这道题修到接近可定稿，我现在不会再走“少选牌就触发额外惩罚”这条路。更合理的方向是改底层规则：

1. 去掉或限制通用外挂资源。
   这一轮验证下来，真正的问题不是“玩家少选了”，而是 `Fire Potion`、`Lantern`、`Block Potion` 这类资源对太多路线都同时变强。
2. 让快解建立在路线专属资源包上，而不是建立在任何牌组都想拿的通用爆发包上。
3. 让慢解建立在专属生存包上，而不是建立在任何牌组都能无脑套用的通用保命包上。
4. 如果教学题为了硬造 `1 回合解` 必须引入通用外挂资源，就应回退为 `2/3/4` 之类更可控的回合梯度，而不是继续强造 `1/2/3`。

## 7. 进一步搜索后的底层结论

我额外用 [difficulty1_variant_search.py](/C:/Users/Administrator/Documents/New%20project%202/difficulty1_variant_search.py) 做了几组规则变体搜索，结论比上一版更明确：

1. 仅仅去掉 `Fire Potion + Lantern` 还不够。
   当资源池里还保留 `Strength Potion` 或 `Block Potion` 这类通用资源时，依然会出现多条 `100%` 稳定路线。
2. `Block Potion` 会把许多原本有风险的慢线抬成稳定 `3 回合解`。
3. `Strength Potion` 会把许多原本需要计算牌序的中速线抬成稳定 `2-3 回合解`。
4. 真正接近可控的方向，是先把这些通用外挂资源拿掉，只靠卡池本身做结构。

进一步结果：

1. 当我把通用药水和通用遗物全部拿掉，只保留卡池本身时，某些变体已经能做到 `7-9 张牌` 审计下 `0` 条 `100%` 稳定路线。
2. 但同时，首回合路线也基本消失了，只剩 `2 回合` 与 `3 回合` 梯度。
3. 这说明：  
   对难度 1 教学题来说，**“必须有 1 回合解”本身就可能和“题目可控”冲突**。

## 8. 当前推进结论

这一轮推进后的有效结论不是“把薄牌惩罚写重一点”，而是：

1. 教学题优先移除通用外挂资源，而不是给少选牌额外惩罚。
2. 资源池要按“路线专属收益”设计，不能按“全路线通用强度”设计。
3. 难度 1 不应机械要求最快解一定是 `1 回合`；在很多情况下，`2/3/4` 会比 `1/2/3` 更可控，也更符合最终目标。

## 11. 新增验证：混合牌会重新制造稳定线

我又额外验证了一件事：  
如果在当前最优近似终稿里，把一张 `Defend` 换成 `Iron Wave`，看起来只是给玩家一点“伤害 + 生存”的温和选择，但结果反而会把题重新推回失控。

验证结果：

1. 当前最优近似终稿（纯 `Strike / Defend / Bash / Perfected Strike`）在 `7-9 张牌` 审计下，`100%` 稳定线为 `0`。
2. 只替换一张 `Defend -> Iron Wave` 后，`100%` 稳定线重新出现，数量回到 `3` 条。
3. 这些稳定线不是靠药水、遗物或薄牌惩罚触发的，而是单纯因为 `Iron Wave` 同时补了输出和生存，压平了原本的路线差。

这说明两件事：

1. 当前真正需要控制的不是“玩家有没有少选牌”，而是“资源里有没有跨路线的双功能外挂”。
2. 对教学题和低难题来说，混合牌和通用药水/遗物一样，都要按高风险资源处理。

1. 伤害密度不足的起手直接把自己锁死在第 2 回合前。
