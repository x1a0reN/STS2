# 技能使用规则

## 必用入口

涉及尖塔残局、卡池、药水、遗物、怪物行动、概率审计、mod 交接时，先用：

- `slay-spire-mod-design-director`
- `slay-spire-puzzle-control`

其中 `slay-spire-mod-design-director` 是总入口，负责玩家视角、设计者视角、mod 制作者视角；`slay-spire-puzzle-control` 是本轮沉淀出的细则 skill，负责可控概率、多解法族、偷跑、难度标准和审计模板。

## 何时用其他技能

| 场景 | 使用技能 |
| --- | --- |
| 用户只给模糊游戏想法，需要先定方向 | `gameplay-design-orchestrator` |
| 输入很乱，需要提取目标、受众、约束 | `gameplay-input-normalizer` |
| 需要定义核心体验和重复决策循环 | `gameplay-fantasy-loop-designer` |
| 需要检查难度曲线、学习曲线、回合压力 | `gameplay-pacing-and-structure` |
| 需要增删卡牌、敌人、状态、资源体系 | `gameplay-system-weaver-and-scope-cutter` |
| 要把题目转成 mod 实现任务、数据表、触发器、测试 | `gameplay-coding-handoff-compiler` |
| 用户要求完整中文制作规格 | `game-design-spec` |
| 已有规格，需要编译成执行计划 | `game-design-execution-compiler` |
| 已有执行计划，需要逐项实现 | `game-design-execution-runner` |

## 安装方式

本包的 `02_可安装技能/skills` 下已经复制了相关 skill 源文件。承接者可以把需要的目录复制到自己的 Codex skills 目录，例如：

```powershell
$src = "本包路径/02_可安装技能/skills"
$dst = "$env:USERPROFILE/.codex/skills"
Copy-Item -Path "$src/*" -Destination $dst -Recurse -Force
```

如果承接环境已经有同名 skill，应先比较 `SKILL.md`，以本包中的 `slay-spire-puzzle-control` 和 `slay-spire-mod-design-director` 为准。

## 每次开工必须读的文件

当用户说“最细致”“可交接”“确认数值”“复核”“偷跑”“难度是否对”时，至少读取：

- `slay-spire-puzzle-control/SKILL.md`
- `references/audit-protocol.md`
- `references/difficulty-scale.md`
- `references/solution-family-and-bypass.md`
- `references/output-templates.md`

如果是在做 D1 或低难度题，再读：

- `references/difficulty1-case-lessons.md`
