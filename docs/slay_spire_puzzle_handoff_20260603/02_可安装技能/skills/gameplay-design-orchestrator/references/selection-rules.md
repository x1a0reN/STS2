# Selection Rules

Use this file to choose one gameplay direction instead of shipping multiple half-decided concepts.

## Generate directions

Generate 2-3 candidate directions at most.

Each candidate must state:

- player identity
- repeatable action
- strongest feeling
- scope pressure
- why it could fail

## Choose one

Pick the strongest direction by these criteria, in order:

1. strongest repeatable fun
2. clearest player promise
3. smallest scope risk for the target scale
4. cleanest path into coding handoff

## Output policy

- Keep only the winning direction in the formal `Gameplay Design Package`.
- Put discarded directions into `02-review-notes.md`.
- For each discarded direction, write one short discard reason.

## Reject weak directions

Reject a candidate if:

- it is mostly theme skin with no new repeatable action
- it needs too many systems for the target scale
- it has no strong first-2-minute retention beat
- it cannot be translated into coding-ready structure
