# 难度 1 实战案例交接记忆

这份记录用于让后续接手者快速继承本轮出题经验。它不是题面文档，而是判断边界和踩坑记录。

## 本轮目标演化

最初目标不是做唯一解题，而是做一道可控的难度 1 残局题。

最终形成的判断是：

- 不追求只有 3 种解法。
- 至少提供 3 个代表解法族。
- 真实解法可以很多。
- 关键是控制解法族、概率区间、最优回合和偷跑上限。

## 难度 1 的实际口径

难度 1 可以是开放式低难度选牌题。

允许：

- 存在多个同族变体。
- 存在比代表解法略强的自然路线。
- 存在较多第 3 回合路线，只要结构相近且可解释。
- 稳线达到 90% 以上。

不允许：

- 100% 无脑稳定线。
- 第 2 回合路线成功率过高。
- 又快又稳的路线压扁全部路线。
- 一眼看出唯一核心牌。
- 文档声称只有三条解法。

## 重要经验

### 完美打击不能独占题目核心

完美打击配大量打击时，玩家很容易一眼看出方向。

处理方法：

- 保留完美打击作为快解或一个解法族。
- 加入平行体系，例如格挡转伤、上勾拳、飞剑回旋、抽牌线。
- 让完美打击不是唯一合理选择。

### 铁斩波风险很高

铁斩波同时给伤害和格挡，和全身撞击组合后容易制造高稳偷跑。

本轮曾出现：

- 铁斩波 + 防御 + 耸肩无视 + 全身撞击
- 第 3 回合高成功率路线
- 直接覆盖主解和稳线

结论：低难度题里如果已有全身撞击，铁斩波要慎用。

### 防御 + 耸肩无视是强生存底座

这两张牌一起出现，会显著提高跨回合稳定性。

如果再加入全身撞击，容易形成：

- 生存稳定
- 输出也够
- 第 3 回合高成功率

这类路线不一定失败，但必须归入预期解法族，不能当作无关干扰。

### 飞剑回旋会抬高快解

飞剑回旋虽然不含“打击”，但它是 1 费 9 点分段伤害。

在完美打击体系里，它可能替代部分打击，抬高第 2 回合路线成功率。

本轮全量审计曾发现：

- 飞剑回旋 + 完美打击 x2 + 打击 x5
- 第 2 回合击杀概率 78.5714%
- 覆盖了原快解 57.1429%

结论：飞剑回旋不是纯干扰牌，它是快解增强牌。

### 上勾拳是强迷惑项

上勾拳纸面强，带易伤，但 2 费会卡能量。

它适合作为迷惑项，但如果同时有抽牌、生存和低费补伤害，也可能进入高成功路线。

## 本轮审计口径

当前题面曾使用：

- 玩家生命：8
- 能量：3
- 抽牌：每回合 5
- 敌人生命：55
- 敌人行动：第 1 回合 0，第 2 回合 9，第 3 回合 11，第 4 回合 13
- 起始牌组：从卡池选择 7-8 张

审计方式：

- 枚举所有合法起始牌组。
- 对每个牌组求最优出牌，不按固定出牌顺序。
- 输出每个牌组在第 1-4 回合的精确击杀概率和失败概率。
- 按首次击杀回合分类。

一次完整审计结果：

- 唯一牌组数：345
- 100% 成功路线：0
- 第 2 回合高于代表快解的路线：5
- 第 3 回合高于代表主解的路线：41
- 第 4 回合高于代表稳线的路线：2

这个结果不自动代表题目失败。它代表文档不能写成“唯一三套解法”，必须写成“代表解法族”。

## 如何判断当前题是否要继续改

如果用户希望的是开放式难度 1：

- 可以接受较多同族解法。
- 重点改文档表述。
- 审计摘要要说明同族变体数量和上界。

如果用户希望的是严格概率分层：

- 需要继续改题。
- 优先处理飞剑回旋抬高第 2 回合的问题。
- 再处理防御 + 耸肩无视 + 全身撞击抬高第 3 回合的问题。

## 推荐后续动作

继续做这道题时，先问或自行判断用户目标：

- 开放式低难度题：保留多解，重写文档为代表解法族。
- 严格三档概率题：继续调卡池和血量，直到越界路线消失。

不要再把“解法数量多”本身当作失败。真正需要控制的是越界路线。

## 本轮形成的最终判断句

可以直接复用：

“这题不是要难到只有三条解法，而是要让所有高成功率路线都能被归入设计者预期的解法族。解法多可以接受；不可接受的是出现又快又稳、绕开核心判断、或让概率梯度失效的路线。”
