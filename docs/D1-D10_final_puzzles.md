# 全局随机性与遍历审计规则（2026-06-09）

核心宗旨：选择资源必须是过关的绝对先决条件，牌序、洗牌、随机目标和随机抽取只能是次要条件。正确资源包可以让玩家获得更高成功率或更早击杀回合，但不能让所有可行解法都变成“等同两张关键牌同手”“必须按极窄顺序抽到”“不管怎么选成功率都明显低下”的纯运气题。

## 题目要义约束

| 维度 | 新规则 |
|---|---|
| 资源优先 | 合法解法族必须先由卡牌、药水、遗物和机制选择区分。牌序只能影响回合分布和小幅成功率，不能成为唯一通关门槛。 |
| 禁止硬同手 | 如果某条主线必须要求两张或多张关键牌同一回合同时上手，否则无法推进核心机制，这条主线不能作为主解。 |
| 禁止低概率主解 | 如果所有主解族在最优策略下成功率都偏低，或最高成功率主要来自少数牌序排列，题目必须重做。 |
| 保留快线 | 允许存在低概率快速击杀线，但同时必须存在成功率更高的中速或慢速稳定线。 |
| 失败原因 | 审计结果必须能区分资源选错、牌序不佳、操作顺序不佳、药水时机错误和随机目标不利，不能只输出一个总成功率。 |

## 枚举方法约束

| 维度 | 新规则 |
|---|---|
| 真实随机态 | 必须枚举起手、自然抽牌、弃牌堆洗回抽牌堆、状态牌加入弃牌堆、随机抽取牌池、随机目标和跨回合保留状态。 |
| 真实资源区 | 资源若按题面可存在于手牌、抽牌堆、弃牌堆、消耗堆或场上状态中，枚举器必须按真实区域建模。不能把“弃牌堆可用资源”错误压成“必须同手上手”。 |
| 最优操作搜索 | 枚举器必须搜索每回合可行出牌顺序、停止出牌时机和药水使用时机。固定出牌顺序只能作为输出排序或 tie-break，不能作为唯一策略。 |
| 解法族报告 | 输出必须包含合法构筑数量、合法资源组合数量、最高成功率、各首次击杀回合最高线、各解法族最高线、稳定线数量和低概率陷阱线。 |
| 运气门槛报告 | 输出必须显式标记是否存在硬同手需求、单一抽牌窗口、随机目标强依赖、随机裂隙强依赖、洗牌后必须立刻抽回等风险。 |
| 发布门槛 | 标记为“待重新枚举”的题，在重新跑完当前规则枚举前，旧概率只能作为结构参考，不能作为最终发布结论。 |

| 维度 | 结论 |
